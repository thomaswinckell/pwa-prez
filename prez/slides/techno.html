Technological needs

  • Small bundle (fast page load)
  • Same code for client and server (except app bootstrap code)
  • Same bundler for client and server (with possibility to have a special configuration)

React

  • JS library for building UI
  • Fast, simple, scalable
  • Support server side rendering

Isomorphic React example

// Client side

import ReactDOM from 'react-dom';

ReactDOM.render(<App/>, document.getElementById('app'))
    
// Server side

import ReactDOMServer from 'react-dom/server';

ReactDOMServer.renderToString(<App/>)

React Router

  • Router library for React
  • Can be used on both client and server side

Isomorphic React-router example

// Client side

import ReactDOM from 'react-dom';
import { BrowserRouter } from 'react-router-dom';

ReactDOM.render(
    <BrowserRouter>
      <App/>
    </BrowserRouter>,
document.getElementById('app'))
    
// Server side

import ReactDOMServer from 'react-dom/server';
import { StaticRouter } from 'react-router-dom';

ReactDOMServer.renderToString(
    <StaticRouter>
      <App/>
    </StaticRouter>
)

Webpack

Webpack

  • Module bundler
  • Flexible, great optimization possibilities
  • Lot of resources, plugins...
  • Allow hot reload on both client and server (with custom configuration of webpack-hot-middleware)
  • Offline plugin (NekR/offline-plugin)
