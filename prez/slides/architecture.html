Architectural needs

  • Small bundle (fast page load)
  • High performance (fast render)
  • Indexable (SEO)
  • Offline mode support

Traditional architecture

Single page app architecture

Isomorphic app

  • Server side rendering then client side rendering
  • Same first render for client and server

Why isomorphic app ?

  • Fast page load and render thanks to SSR
  • Works without JS thanks to SSR (at least enough for SEO)
  • Old devices support
  • Flexibility of CSR
  • Easier offline mode support (don't need to request API or put API requests in cache to have something viewable)
  • Same code for client and server (except app bootstrap code)
